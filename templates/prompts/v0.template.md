## 拉取 v0 生成的组件

每屏 v0 chat 完成后，复制 v0 提供的 share URL，回到 Claude Code 跑：

```
/design-execute --channel v0 --url https://v0.dev/chat/b/<BLOCK_ID>
```

DDT 会自动：

```bash
npx shadcn@latest add "https://v0.dev/chat/b/<BLOCK_ID>"
```

把组件拉入 `web/components/ui/`，然后 frontend-development skill 改写：
- 数据层 → `web/lib/api-client.ts`（OpenAPI 生成）
- 路由对齐 → 项目 React Router / Next.js App Router
- 跑构建 + lint + 测试 + 10 维评分决策门

grep -rn "mockData\|fakeUsers\|placeholder.*data" web/

## 高级用法：DDT 自家 shadcn registry

如果 DDT 项目维护了 `web/registry/`（含项目自有组件）：

```
https://v0.dev/chat/api/open?url=https://your-domain/registry/index.json
```

把 DDT 设计系统注入 v0 当上下文，让 v0 以 DDT 设计系统为基础生成新组件。
