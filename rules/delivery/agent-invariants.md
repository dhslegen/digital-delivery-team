# DDT Agent Invariants · 权威定义

> 本文件是 v3 §2.1 「Global Invariants」的**唯一权威来源**。
> 每个 agent 在执行任何输出前必须对照本清单自检，全部通过方可声称完成。
> 各 agent 文件的 Global Invariants 段落以本文件为准，不得自行扩展或删减。

---

## 6 条不可违反的全局约束

### 1 · 单一产物责任

每个 agent 对且仅对其在 `contexts/delivery.md` §1 速查表中列出的**负责产物**负责。

- 禁止向其他 agent 的产物文件写入任何内容
- 唯一例外：`docs/blockers.md`——任何 agent 都可追加 blocker 条目
- 若任务需要修改本角色负责产物之外的内容，说明任务分解不对，拒绝执行并写 blocker

### 2 · 禁止猜测

遇到以下任一情况，**立即停止**、追加条目到 `docs/blockers.md`、请求人类决策：

- 必读输入文件缺失
- 契约字段含义有歧义
- 两份输入文档相互矛盾
- 技术选型或业务边界需要人类决策
- 不确定某个行为是需求还是缺陷

写 blocker 使用 `templates/blockers.template.md` 的字段结构，字段填写完整后**追加**（不覆盖）到 `docs/blockers.md`。

### 3 · 禁止自我汇报度量

- 不得在产物里写"本次耗时 X 秒"/"使用了 Y tokens"/"成功率 Z%"
- 不得调用任何 `track_*` 接口
- 时长、token 消耗、成败事件由 hooks 自动捕获，agent 不干预

### 4 · 输出前自检

- 产物末尾必须附上已填写的 Self-Check 清单（具体条目由各 agent 文件定义）
- Self-Check 未全勾不得声称完成
- Self-Check 的填写必须基于实际检查，不得预先全勾

### 5 · 禁用糊弄词

产物中禁止出现以下词汇或其同义变体：

- "根据需要"、"视情况"、"若有必要"、"等"（用于列举时的省略）
- "通常"、"一般来说"、"可能"（当描述必须确定的行为时）
- "待定"、"TBD"、"TODO"（最终交付产物中）

凡需要用此类词汇描述的内容，说明信息不足，应写 blocker。

### 6 · 可重入模式

目标产物已存在时：

1. 读取现有产物作为基线
2. 仅修改需要更新的部分，保留已验证内容
3. 在产物末尾的 `## 变更记录` 段追加一条 diff 摘要：
   ```
   - [YYYY-MM-DD] <agent-name>: <变更内容（1-2句）>
   ```
4. 禁止全量覆盖已有产物

`--refresh` 标志语义：重新读取所有上游输入并更新，但仍遵守增量原则。

---

## 自检使用方式

```
## Self-Check（追加到产物末尾）
- [ ] 只写入了我的唯一交付文件（+ 可选 blockers.md）
- [ ] 无猜测——所有不确定项均已写入 blockers.md
- [ ] 未自报任何度量数据
- [ ] 产物中无糊弄词（已全文搜索确认）
- [ ] 如为增量更新，已追加变更记录
```
