## 复杂度判定标准

| 复杂度 | 判定条件 |
|-------|---------|
| **简单** | endpoint ≤ 5；无第三方集成；数据模型 ≤ 3 个实体 |
| **中等** | endpoint 6–15；≤ 1 个第三方集成；实体 4–8 个 |
| **复杂** | endpoint > 15；多个第三方集成；实体 > 8 个；高并发要求 |

## 任务粒度规则（供 WBS 拆分参考）

| 任务类型 | 推荐粒度(h) | 拆分触发条件 |
|---------|-----------|-----------|
| 单个 API endpoint 实现 | 1–2h | 含复杂业务逻辑则拆为 2 个任务 |
| 单个页面/组件 | 1–3h | 含复杂状态管理则拆分 |
| 数据库迁移 | 0.5–1h | 每个 migration 文件独立任务 |
| 单元测试编写 | 0.5–1h / 模块 | — |
| 集成测试编写 | 1–2h / endpoint | — |
| 文档编写 | 1–3h | — |

## 注意事项

1. 以上系数基于单人全栈工程师 + AI 辅助场景，多人团队需重新校准
2. 首次跑新类型项目（如 mobile / data pipeline），建议完成后将数据追加到 `historical-projects.csv`
3. 若实际工时超出基准 ×1.5，视为异常，需记录原因到项目备注
